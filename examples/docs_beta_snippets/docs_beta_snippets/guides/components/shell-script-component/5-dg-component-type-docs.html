Component: my_component_library.lib.ShellCommand

Description:

Models a shell script as a Dagster asset.

Component Schema:

  • attributes - object
    Subfields
    • script_path - string
    • asset_specs - [object]
      Subfields
      • deps - [string]
      • description - string | null
        Human-readable description of the asset.
        Example: "Refined sales data"
      • metadata - string
        Additional metadata for the asset.
      • group_name - string | null
        Used to organize assets into groups, defaults to 'default'.
        Example: "staging"
      • skippable - boolean
        Whether this asset can be omitted during materialization, causing downstream dependencies to skip.
      • code_version - string | null
        A version representing the code that produced the asset. Increment this value when the code changes.
        Example: "3"
      • owners - [string]
      • tags - string
        Tags for filtering and organizing.
        Example: { "team": "analytics", "tier": "prod" }
      • kinds - [string]
      • automation_condition - string | null
        The condition under which the asset will be automatically materialized.
      • key - string
        A unique identifier for the asset.
