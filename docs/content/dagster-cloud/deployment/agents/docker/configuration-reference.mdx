# Docker agent configuration reference

<Note>This guide is applicable to Dagster Cloud.</Note>

This reference describes the various configuration options Dagster Cloud currently supports for [Docker agents](/dagster-cloud/deployment/agents/docker/configuring-running-docker-agent).

---

## Environment variables and secrets

Using the `container_context.docker.env_vars` property, you can include environment variables and secrets in the Docker container associated with a specific code location. For example:

```yaml
## dagster_cloud.yaml

container_context:
  docker:
    env_vars:
      - DATABASE_NAME
      - DATABASE_USERNAME=hooli_testing
```

The `container_context.docker.env_vars` property is a list, where each item can be either `KEY` or `KEY=VALUE`. If only `KEY`, the value will be pulled from the local environment.

**Note**: Environment variables and secrets can also be set using the [Dagster Cloud UI](/dagster-cloud/developing-testing/environment-variables#declaring-environment-variables).
