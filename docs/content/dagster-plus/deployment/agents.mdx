# Dagster+ Hybrid agents

For [Hybrid deployments](/dagster-plus/deployment/hybrid), Dagster+ uses an agent that is responsible for executing your code. The agent streams metadata about code execution over HTTPS to Dagster+â€™s Agent API.

Dagster+ currently supports agents running on the following backends:

<table
  className="table"
  style={{
    width: "100%",
  }}
>
  <thead>
    <tr>
      <th
        style={{
          width: "30%",
        }}
      >
        Name
      </th>
      <th>How it works</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <a href="#amazon-ecs-agents">Amazon Elastic Container Service (ECS)</a>
      </td>
      <td>
        The Amazon Elastic Container Service (ECS) agent executes Dagster jobs
        as Amazon Web Services (AWS) ECS tasks. This agent is appropriate for
        scaled production deployments; ECS is a good choice for teams who have
        already standardized on ECS or who don't plan to run their own container
        orchestration infrastructure.
      </td>
    </tr>
    <tr>
      <td>
        <a href="#docker-agents">Docker</a>
      </td>
      <td>
        The Docker agent executes Dagster jobs in Docker containers on your
        computer.
      </td>
    </tr>
    <tr>
      <td>
        <a href="#kubernetes-agents">Kubernetes</a>
      </td>
      <td>
        The Kubernetes agent executes Dagster jobs on a Kubernetes cluster. This
        agent is appropriate for scaled production deployments and is a good
        choice for teams who have already standardized on Kubernetes.
      </td>
    </tr>
    <tr>
      <td>
        <a href="#local-agents">Local</a>
      </td>
      <td>
        The local agent executes Dagster jobs as processes on your computer.
      </td>
    </tr>
  </tbody>
</table>
